Анализ лога установки пакетов (Ubuntu)
=====================================

Критичная ошибка
----------------
- **dpkg: pre-dependency problem — libpam-modules не устанавливается.**
  - Причина: новая версия `libpam-modules` (1.5.3-5ubuntu5.5+bot1) требует точно такую же версию `libpam-modules-bin`, но в системе была сконфигурирована более старая 1.5.3-5ubuntu5.4.
  - Как исправить:
    1. Сначала обновить `libpam-modules-bin` до 1.5.3-5ubuntu5.5+bot1.
    2. Затем переустановить `libpam-modules` той же версии.
    3. Для автоматического завершения зависимостей можно выполнить `apt-get -f install` или установить оба пакета одной командой `apt install ./libpam-modules_1.5.3-5ubuntu5.5+bot1_amd64.deb ./libpam-modules-bin_1.5.3-5ubuntu5.5+bot1_amd64.deb`.

Предупреждения
--------------
- **update-alternatives: не создана ссылка для man-страницы automake** — отсутствует `automake-1.16.1.gz`. Это не влияет на работу; при необходимости установить/проверить пакет документации.
- **dpkg-divert: перенос `/usr/bin/tar` из пакета Essential** — опасно переименовывать базовую утилиту. Лучше использовать `--no-rename` или избегать диверсии `tar`.
- **pip под root** — ставьте пакеты в виртуальное окружение или с `--user`, чтобы избежать конфликтов с системными пакетами.
- **cp: опция -n непереносима** — замените `cp -n` на эквивалент `cp --update=none`.
- **apt-key устарел** — перенесите ключи репозиториев из `/etc/apt/trusted.gpg` в `/usr/share/keyrings/` и укажите `signed-by` в `.list` файлах.
- **debconf: apt-utils не установлен** — в неинтерактивной среде можно игнорировать либо установить `apt-utils`.
- **APT скачивает без sandbox (_apt)** — убедитесь, что каталог загрузки доступен пользователю `_apt`, чтобы не скачивать файлы под root.

Краткое резюме
--------------
Главная проблема — несогласованные версии PAM: сначала обновите `libpam-modules-bin`, затем `libpam-modules`, либо доверьте порядок `apt-get -f install`. Остальные предупреждения некритичны, но их устранение улучшит безопасность и переносимость сборок.
